* **Emoji Accessibility:** Any emoji used in the UI must be wrapped in a `<span role="img" aria-label="...">` tag. Screen readers often struggle with bare emojis, especially when they are used as core visual identifiers for gameplay elements (e.g. 🍕 and ☕ for points).
* **Device-Specific Instructions:** To avoid confusing mobile users with keyboard instructions, utilize `.desktop-only` and `.mobile-only` utility classes (toggled via `@media (pointer: coarse)`) to show the correct control hints in `#overlay` and `.control-legend`.
